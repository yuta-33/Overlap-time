# overlap-time MVPスコープ確定（Step 1）

## 1. 目的
- 「When2meetっぽいが、日本人が迷わず使える」予定調整を、無料で最短リリースする。

## 2. 今回のMVPで実装する（In Scope）
- イベント作成
  - イベント名
  - 日付範囲（開始日〜終了日）
  - 1日の表示時間帯（例: 09:00-24:00）
  - スロット粒度（15分 or 30分）
- 参加リンク共有（ログイン不要）
- 参加者入力（表示名のみ）
- 空き時間入力（2値）
  - `空き` / `不可`
  - ドラッグまたはスワイプで連続入力
- 集計表示（ヒートマップ）
  - 濃さ = そのスロットを `空き` とした人数
- 日本語UI最適化（最低限）
  - `Asia/Tokyo` 固定
  - 日付・曜日の日本語表示
- 最低限の悪用耐性
  - 推測困難ID（nanoid）
  - 軽いレート制限
  - XSS対策（表示名のエスケープ）

## 3. 今回は実装しない（Out of Scope）
- 3段階評価（`〇/△/×`）の `△`
- コメント機能
- 通知機能（メール/Push）
- 外部カレンダー連携（Google/Apple等）
- アカウント認証
- 祝日データ連動
- 候補時間の自動提案ボタン（上位3件など）

## 4. 技術方針（MVP時点）
- フロント: Next.js
- バックエンド/DB/Realtime: Supabase
- デプロイ: Vercel（無料枠）

## 5. MVP完了条件（リリース判定）
- イベント作成から共有リンク発行まで完了できる。
- 複数参加者が同一イベントに参加し、空き/不可を更新できる。
- 集計ヒートマップが更新を反映する。
- スマホで基本操作（入力・スクロール・保存）が破綻しない。
